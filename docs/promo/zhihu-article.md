# 我做了一个把 AI 写作提示词全部开放给用户的工具——故事熔炉 StoryForge

> **GitHub 开源免费，纯本地运行，11 家 AI 厂商任选，提示词全透明可改。**

---

## 先说一个让我很烦的问题

用过市面上那些 AI 写作工具的朋友，大概都有过这种体验：

你点下「AI 续写」，等了几秒，出来了。但出来的东西**不是你想要的风格**。你改了几个字，又点，还不是。你开始怀疑：这个 AI 到底接收到了什么指令？为什么它总在生成那种千篇一律、"AI 味"十足的段落？

你搜不到答案，因为提示词在后台的黑箱里。

---

## 我的解法：把黑箱打开

**故事熔炉（StoryForge）** 的核心理念只有一句话：

> **每一个 AI 按钮背后的 System Prompt + User Template，全部可见、可编辑、可保存。**

这不是噱头。内置了 **43 条提示词模板**，你打开任何一条，能看到：

- 完整的 System Prompt（不是"请帮我续写小说"这种废话，而是精心设计的角色扮演指令）
- 带有 `{{变量}}` 占位符的 User Template
- 每个可调参数（基调、详尽度、字数、视角……）
- 示例（👍 好样本）和反例（👎 坏样本）——这些自动拼接到 prompt 末尾，真正的 few-shot

你不喜欢某条内置模板？**一键「克隆为我的版本」**，改 system prompt、改参数定义、改示例，改完设为激活，下次点那个 AI 按钮就用你的版本。

## 功能全景（认真看，比较长）

### 一、著作信息

新建项目之后，首先是项目基础信息：

- **77 个流派标签**（按起点/纵横/晋江三平台分类，仙侠/武侠/悬疑/言情/都市……全覆盖）
- 目标字数（最高 500 万字，认真的）
- 创作状态：构思 / 连载 / 暂停 / 完结
- 创作统计：连续写作天数、最近更新时间

还有一个**参考书目**模块。你可以添加「故事参考」和「风格参考」，AI 生成时这些书目会自动作为隐式风格指引传入 prompt。想让 AI 写出《白鹿原》那种气质？把它加进参考书目。

---

### 二、设定库（最重）

#### 世界观 · 三子模块

**世界起源**（3 个字段，每个都可以单独 AI 生成）：
- 世界来源：创世神话/科技起源等
- 力量层次：修真等级/魔法体系/科技层级
- 神明设定：存在性、层级、名号、规则

**自然环境**（11 个字段）：
世界结构、大陆分布、区域面积、山川河流、分区域气候，自然资源还细分了四个子字段（珍禽异兽/灵药/矿石/特产）。

**人文环境**（7 个字段）：
世界历史线、大事记、种族、势力分布、政治经济文化、矛盾冲突、道具设计。

**智能上下文**：点 AI 生成任意字段时，你已经填好的其他字段会自动作为 context 传入，保证内部逻辑自洽。世界起源写了"宇宙大爆炸说"，生成自然环境时 AI 不会再给你安排神明创世。

---

#### 角色设计 · 四级分档

| 等级 | 视图 | 适用场景 |
|------|------|----------|
| 主角/重要配角 | 大卡片完整表，全字段 + AI 设计 | 弧光完整的核心人物 |
| 次要角色 | 小卡片 3 列网格 | 有戏份但不主导的配角 |
| NPC | 紧凑列表（姓名+地点+一句话）| 常驻不出场角色 |
| 路人 | 5 列表格（姓名/出场/章节/作用/结局）| 群像一笔带过 |

每级角色的信息密度是不同的。主角要什么背景、欲望、创伤、弧光、说话方式全都要写；路人只需要一个名字和一行说明就够了。这个分级设计极大降低了角色管理的心智负担。

另有**关系网**模块：D3-force 力导图，支持亲属/师友/敌对/暧昧/合作/竞争六类关系，可视化整个角色关系网络。

---

#### 故事设计

7 个字段，全部可 AI 生成：
- 一句话故事（logline）
- 故事概念
- 主题
- 核心冲突
- 故事模式（线性/莲花地图/多线并行/蒙太奇）
- 主线
- 复线

### 三、创作区 · 六模块

| 模块 | 核心功能 |
|------|----------|
| **创作规则** | 写作风格/叙事视角（一/三/全知/多视角）/基调/禁忌/一致性规则 |
| **大纲** | 卷/章节树，AI 生成 3-5 卷概要，AI 展开 15-25 章/卷，参数滑块控制节奏/卷数 |
| **细纲** | 单章拆 3-6 场景（标题/概要/人物/地点/冲突/节奏/字数估算） |
| **章节列表** | 跨卷扁平视图，状态徽章（仅大纲/初稿/已修改/已润色/定稿），字数统计 |
| **正文编辑器** | TipTap 富文本，自动保存，**5 个 AI 操作**（写正文/续写/润色/扩写/去 AI 味） |
| **伏笔** | 10 类型（契诃夫之枪/预言/象征…），4 状态流，AI 批量建议，列表+看板双视图 |

## 三、提示词系统（这个真的是核心）

### 调参浮窗——每个 AI 按钮都有

点任何一个「AI 生成」按钮，右下角弹出 PromptRunPanel 浮窗：

```
┌─────────────────────────────────────┐
│ 📋 当前模板：内置-世界观维度生成      │
│                                     │
│ 基调:    [严肃 ▾]     [☑ 启用]      │
│ 详尽度:  [中等 ▾]     [☑ 启用]      │
│ 字数估算: [800 ↕]    [☑ 启用]       │
│                                     │
│ ▾ 高级：临时修改 Prompt 文字          │
│   (不写回模板，只影响本次生成)        │
│                                     │
│ [⟲ 重置]  [💾 另存为我的版本]        │
│                                     │
│ ──── 流式输出 ────                   │
│ ...AI 正在生成...                    │
│                                     │
│ 👍 标好示例  👎 标反例  ✓ 采纳写入   │
└─────────────────────────────────────┘
```

三层调整粒度：
1. **参数滑块/下拉**：最快，调完直接影响结果
2. **临时文字覆盖**：直接改这次的 prompt 文字，不影响模板本体
3. **「另存为我的版本」**：把调好的设置保存为新模板

生成结果满意，点 👍 标为好示例——下次 AI 生成时这条示例会自动拼接进 prompt。不满意，点 👎 标为反例——AI 会主动避开这种写法。**少则 3-5 个 few-shot 积累下来，生成质量会有明显跃升。**

---

### 题材包热切换

内置 **5 套题材包**，每套 6 条专属提示词：

| 题材包 | 风格特点 |
|--------|----------|
| 通用（19 条）| 均衡全模块覆盖 |
| 仙侠修真 | 飞升体系、典雅文笔、心境优先 |
| 言情 | 心理戏、CP 张力、双视角 |
| 现实主义 | 日常感、克制、去套路化 |
| 悬疑推理 | 信息控制、不可靠叙事、章末反转 |

**热切换**：顶部下拉选一个题材包 → 后台批量切换所有关联模板 → 回到创作区，AI 风格立刻改变。

同一个项目里，30 秒内可以试 5 种完全不同的写作风格。适合在初稿期快速探索哪种风格最对味。

## 四、工作流引擎

这个功能是写长篇的利器。

工作流 = 一组有顺序的 AI 步骤，每步可以自动把结果写回项目对应字段。

**3 条内置工作流**：

| 工作流 | 步骤 | 适用场景 |
|--------|------|----------|
| **极速起书·通用** | 故事核心→世界起源→主角→卷大纲→第一章 | 从零起书，一键跑出框架 |
| **单章深度生成** | 拆场景→写正文→润色→去 AI 味 | 精雕细琢一章 |
| **伏笔体系搭建** | 世界观摘要→伏笔建议 | 中期补伏笔 |

每一步都是独立卡片，有状态色（灰/蓝/绿/红），可以重新生成/跳过/复制/手动保存。

**saveTarget 自动写回**特别好用：步骤跑完不需要你手动复制粘贴，AI 输出的角色列表直接批量写入角色表，大纲节点直接写入大纲树，伏笔直接写入伏笔库。

用户也可以**自建工作流**，导出 JSON 分享给其他人。

## 五、大师研读系统（最新功能）

这个功能的逻辑是：**先读好书，再写好书。**

你上传一部你欣赏的小说，系统自动分块，从五个维度深度分析：

| 维度 | 分析内容 |
|------|----------|
| 🌍 世界观 | 世界构建方法、设定层次、信息投放节奏 |
| 👤 角色 | 人物塑造手法、弧光设计、对话风格 |
| 📖 情节 | 叙事结构、冲突设计、节奏控制 |
| 🔮 伏笔 | 伏线铺设手法、呼应方式、信息隐藏技巧 |
| ✍️ 文笔 | 修辞手法、句式节奏、风格指纹 |

三级分析深度可选：
- **快速**：~4 万字/块，概览全书脉络
- **标准**：~2.5 万字/块，平衡效率和细节
- **深度**：~1.5 万字/块，逐章精读拆解

分析结果存在独立的数据表里，**不污染你的创作数据**。可以跨多部作品归纳，按流派筛选，逐渐形成你自己的方法论库。

## 六、百万字级文档导入

有时候你已经在别处写了很多内容，或者有一份很长的参考文献，想导入进来。

**基础导入**：粘贴文字或上传 `.txt/.md/.csv`，选类型（角色文档/世界观文档/大纲文档），AI 解析 → JSON 预览 → 确认 → 批量入库。

**分块导入流水线**（百万字级）：

这个是真正的工程级方案：

1. 上传文件 → SHA-256 去重（防重复导入）
2. 智能分块（按章节/段落边界切割，不会在句子中间断）
3. 原文 Blob 持久化到 IndexedDB（关闭浏览器不丢）
4. 逐块 AI 解析（并发受控，失败自动重试 3 次）
5. Jaccard 相似度角色去重合并（同一角色跨分块不重复）
6. 批量写入数据库

支持**暂停/恢复**：中途可以暂停，下次打开浏览器会自动检测未完成任务，一键恢复。

## 七、导出与数据安全

**5 种导出方式**：

| 方式 | 用途 |
|------|------|
| JSON | 完整备份，可在另一台设备完整恢复 |
| Markdown | 结构化文档，适合存档分享 |
| TXT | 纯文本，按卷章顺序，适合投稿发布 |
| File System Access | 直接写入本地文件夹（需 Chromium） |
| GitHub Gist | 云端备份，支持增量更新 |

**版本历史**：每 5 分钟自动快照（保留 20 条），支持手动打「永久快照」，时间线视图。恢复时不覆盖当前项目，而是创建新项目（双保险）。

## 八、AI 接入（11 家厂商）

| 分类 | 厂商 |
|------|------|
| 国际 | OpenAI / Anthropic Claude / Google Gemini / DeepSeek |
| 国内 | 通义千问 / 豆包 / 文心一言 / Kimi / 智谱 GLM / MiniMax |

想用便宜的 DeepSeek 打草稿，用 Claude 做精修？随时切换，不用重新配置项目。

## 最后

AI 写作工具这条路，我认为走向应该是：**让作者越来越懂 AI，而不是越来越依赖 AI**。

提示词全透明、可编辑、可积累——这是 StoryForge 和大多数 AI 写作工具最根本的区别。

你调好的模板，下次还在。你标的好示例，会越积越准。你理解了提示词怎么工作，AI 就从一个玄学黑箱变成了你的创作协作者。

**创作的快感不该被 AI 抹掉，而是被 AI 放大。**
