# 一致性覆盖地图 + 反例基线(诚实现状 · 供审查)

> **目的**:用可证伪的方式,如实标出"当前每一类矛盾,是代码**硬检测** / LLM **软检测** / **完全没检测**",作为一致性工程化的真实起点与度量基线。
> **为什么有这份**:根治"把'代码跑通 / 上下文装配成功'冒充'内容一致'"的过度承诺(`CLAUDE.md` 反复警告的"自己说修了实际无效")。实例:曾声称"上下文能保持一致",实则**世界起源 ↔ 力量体系互斥**。
> **状态**:Claude 基于代码逐条核查起草 2026-07-10。**待作者 + GPT 外部审查**——本文件一切结论都附代码依据,可逐条查。

---

## 0. 一句话现状(不粉饰)

经代码核查:整个项目**确定性的"矛盾判决"函数实际上只有一个** —— `checkHeldItemAcquisition`(物品重复获得)。其余一致性能力,要么是"检索期过滤 / 结构写回校验 / 状态最新覆盖"(有用,但**不判语义矛盾**),要么是 **LLM 软审校(只提示、不阻断、会漏)**。**"设定互斥"那一类(世界起源↔力量体系),零代码覆盖,只有提示词里一句"请保持一致"。**

**两个维度必须分清(GPT 审查请重点看这里):**
- **检测可靠性**:🟢硬(代码确定性检出,不漏)/ 🟡软(LLM 检出,会漏会误)/ 🔴无(没有任何检出)。
- **处置强度**:目前**所有检测结果一律是 advisory(给作者看),没有一处 blocking**。连 held-items 的确定性检出,也只是流向 ReviewPanel 提示作者,不阻断生成。所以现在连"硬检测"也只做到"**不漏地告诉你**",没做到"**强制不让写崩**"。

| 认知边界 / 开天眼(角色表现出知道未获知信息) | 无 | 无(`CONSISTENCY-2` 认知账本才补) | 🔴无 | 无 |

## 3. 方法论(杜绝再次过度承诺)

1. **任何一致性声明 = 三段式**:"🟢 这 N 类我硬检测(附反例测试)+ 🟡 这 M 类只有软提示 + 🔴 这 K 类没覆盖"。**不许只说"一致了"**。
2. **绿灯只对它测的那件事有效**:"代码跑通 / 上下文装配成功"是一回事,"内容一致"是另一回事,不得互相冒充。
3. **覆盖地图是活文档**:每块 `CONSISTENCY-*` 砖落地,必须回来更新第 1 节等级 + 第 2 节考卷。
4. **硬检测 ≠ 硬处置**:可另行决定把哪些 🟢硬检测升级为 blocking / 强制作者确认(现在全是 advisory)。

## 4. 补最狠那一类的设计:世界宪法 + 设定一致性校验(针对 🔴 设定互斥)

**问题**:散文设定之间的语义冲突,代码一般判不了(自然语言无法确定性解析任意散文)。

**思路(软→硬,和 held-items 精神一致)**:
- **抽取(软 · LLM + 作者确认)**:把关键设定**断言**从散文抽成**结构化 canonical assertion**,例:`{ 主题: "魔法来源", 值: "月亮潮汐", 来源: worldOrigin#123 }`。作者确认 = 软→硬闸门。**这批已确认断言 = 世界宪法(canon)**。
- **比对(硬 · 代码)**:新设定 / 正文落库或生成前,把它涉及的断言与世界宪法比对——**同主题、不同值 = 冲突,确定性硬标**。
- **诚实的覆盖边界**:只有**已被抽成断言并经作者确认**的设定,才进入 🟢 硬保证;没抽的散文仍 🟡/🔴。**覆盖随抽取增长,永远不到 100%**——但重要的、已声明的设定变可保证。这是这类问题唯一诚实的做法,别指望"全自动全覆盖"。

**四问 checklist**:读走 `assembleContext`(新增 `canonAssertions` 源);断言库新表 → 先进 `PROJECT_TABLES`(owner/worldScoped/refs/exportable);写走 `adopt()`。数据红线:新表 = 迁移测试 + 导出/导入往返。

→ 落 ROADMAP 为 **`CONSISTENCY-3 · 世界宪法 + 设定一致性校验`**(收敛路线第 4 步的一块)。

## 5. 结论(给作者和 GPT 审查)

- **现状是"一块硬砖(held-items)+ 一层会漏的 LLM 软审 + 大片裸奔"**,且全 advisory、无 blocking。你之前的不信任,数据支持。
- **一致性工程化的真实含义**:不是"开关一开就一致",而是"**把一致性从 100% 靠祈祷,一类一类地搬进代码硬保证**",每搬一类更新这张地图。**永远会有 🔴/🟡 残留**——诚实的产品是把地图摆给用户看,而不是假装全绿。
- **先做这张地图本身,就是最该先做的一步**:它是度量基线,也是防止我(或任何接手 AI)再次过度承诺的制度防线。
