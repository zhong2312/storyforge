# StoryForge 故事熔炉 · 如何打开项目

> 给完全没接触过代码的朋友。跟着一步步做就行。
> **不想折腾的，直接看「方式一」用网页版，不用下载任何东西。**

---

## 方式一：直接用网页版（最简单，强烈推荐）

打开浏览器，访问这个网址：

### 👉 https://yuanbw.vercel.app/storyforge/

打开就能用，不用下载、不用安装、不用敲任何命令。功能和本地版**完全一样**。

数据存在你自己的浏览器里（不上传服务器）。换电脑/换浏览器不会自动同步，重要内容请用工具里的「导出」存成文件备份。

> **大部分人到这里就够了。** 下面的「本地运行」是给想离线用、或想自己改代码的人看的，步骤会多一些。

---

## 方式二：在自己电脑上本地运行

整个过程就两件事：**① 装一个叫 Node.js 的运行环境（只装一次）→ ② 在项目文件夹里敲两行命令启动**。

下面讲得很细，别被吓到，照做即可。

---

### 第 1 步：安装 Node.js（一台电脑只需装一次）

1. 浏览器打开 👉 https://nodejs.org/zh-cn
2. 点页面上的 **LTS** 版本下载（LTS = 长期支持版，更稳定）
3. 双击下载好的安装包，**一路点「下一步 / Next」直到完成**
   - Windows 安装过程中如果看到 **"Add to PATH"** 的勾选项，确保它是打勾的（默认就是打勾的，一般不用管）
4. 装完**重启一下电脑**（保险起见，能避免后面命令找不到的问题）

---

### 第 2 步：下载项目并解压

1. 浏览器打开 👉 https://github.com/yuanbw2025/storyforge
2. 找到绿色的 **「Code」** 按钮，点它 → 点 **「Download ZIP」**
3. 下载完是个压缩包，**右键解压**到一个你找得到的地方（比如桌面）
4. 解压后你会得到一个叫 **storyforge** 的文件夹，**记住它在哪**

---

### 第 3 步：在「项目文件夹里」打开命令窗口 ⭐（最关键，很多人卡在这）

命令不能随便找个地方敲，**必须在 storyforge 这个文件夹"里面"敲**。最不容易出错的方法如下：

#### Windows：

1. 用「文件资源管理器」**双击进入 storyforge 文件夹**（进到里面，能看到 `package.json`、`src` 这些文件）
2. 看文件夹窗口**最上面那条显示路径的横条**（地址栏，比如显示 `桌面 > storyforge`）
3. **鼠标左键点一下那条地址栏的空白处**，它会变成可编辑的文字
4. 把里面内容全删掉，**打字输入 `cmd`，然后按回车**
5. ✅ 这时会弹出一个**黑色的命令窗口**，而且它已经"站在"你的项目文件夹里了——这就对了

> 也可以：在文件夹空白处**按住 Shift 键 + 鼠标右键** → 选「在此处打开 PowerShell 窗口 / 终端」。但不同 Windows 版本菜单不一样，上面的「地址栏输 cmd」最通用。

#### Mac：

1. 打开「访达」，进入 storyforge 文件夹
2. 顶部菜单「访达」→「服务」→「新建位于文件夹位置的终端窗口」
   （如果没有这个选项：打开「终端」App，输入 `cd ` 然后空一格，把 storyforge 文件夹**拖进终端窗口**，按回车）
3. ✅ 出现的终端窗口就"站在"项目文件夹里了

---

### 第 4 步：装依赖（只需做一次）

在刚才打开的黑色命令窗口里，**打字输入下面这行，然后按回车**：

```
npm install
```

然后**等它跑完**（屏幕会刷一堆字，约 1~2 分钟，需要联网）。看到命令窗口重新能输入了、没报红色 error，就成功了。

> **如果卡住、报错、或一直装不上**（国内网络常见），先输入下面这行换成国内下载源，再重新 `npm install`：
> ```
> npm config set registry https://registry.npmmirror.com
> ```

---

### 第 5 步：启动

接着在同一个命令窗口里，**输入这行，按回车**：

```
npm run dev
```

看到屏幕出现类似 `Local: http://localhost:5175/...` 的字样，就启动成功了。

---

### 第 6 步：打开浏览器看效果

打开浏览器，访问：

### 👉 http://localhost:5175/storyforge/

（很多时候浏览器会自动打开；没自动打开就手动把这个地址复制进浏览器。）

---

### 以后再用怎么办？

- **不用再装 Node、也不用再 `npm install`**（那俩只需第一次）
- 只要：进 storyforge 文件夹 → 地址栏输 `cmd` 回车 → 输入 `npm run dev` 回车 → 开浏览器
- **那个黑色命令窗口不要关**，关了就停止运行了；用完直接关窗口即可停止

---

## 方式三：用 Trae / VS Code 这类软件打开

如果你会用 Trae（AI 编程软件）或 VS Code：

1. 软件里「打开文件夹」选中解压后的 storyforge
2. 打开软件自带的「终端」
3. 在终端里同样输入 `npm install`（第一次）→ `npm run dev`，再开浏览器

> Trae 可以让 AI 帮你执行命令，对不熟命令的人友好。但底层和方式二一样，都是这两行命令。

## 常见问题

**Q：以前的 start.bat 双击没反应 / 一闪而过 / 报错？**
A：start.bat 只能 Windows 用，还依赖系统环境，容易因为没装 Node、被杀毒软件拦、或网络问题失败。**推荐改用方式二**手动敲命令，更稳、Win/Mac 都能用、永久有效。

**Q：输入命令后提示 `npm 不是内部或外部命令` / `command not found`？**
A：说明 Node.js 没装好。回到第 1 步重装，装完**重启电脑**，再重新打开命令窗口试。

**Q：我是不是在错的地方敲命令了？**
A：命令窗口必须"站在" storyforge 文件夹里。判断方法：黑窗口里那行路径结尾应该是 `\storyforge>`（Windows）。如果不是，重做第 3 步。

**Q：端口 5175 被占用？**
A：关掉占用的程序；或它会自动换端口，按命令窗口里实际显示的地址打开。

**Q：我的小说会丢吗？**
A：数据存在浏览器本地，不上传服务器，**不清浏览器数据就一直在**。换电脑/浏览器不会同步，重要内容请用工具内「导出」备份。
